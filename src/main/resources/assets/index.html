What is Alertz?

This alerting platform allows users to identify breaches in various metrics(system, operation and business) across systems and escalate alerts via nagios or mail.

How Does Alertz Work?

Alerts can be generated for breaches in trends on an archived as well as real time data. Currently Alertz supports graphite and opentsdb for archived metrics but it can be easily extended to support other data sources as well. Support for other data sources can be added by writing plugin clients for various metric sources.

At its core, Alertz comprises of a rule engine that lets you run checks based on the metrics fetched from a data source.

There are two kinds of rules:

  1. Scheduled Rules: These are rules that need to be run at regular intervals on archived data.
  2. OnDemand Rules: These are rules that are run only when required. In this mode, the application triggers the rule by publishing real time metrics to the http end point (onDemand Rule end point) exposed by Alertz.
Modes of Notification

Currently, Alertz provides two kinds of notification

  1. Push/Subscription Model: Users can subscribe to notifications from Alertz. Currently, we support email and nagios (With full fledged escalation).
  2. Pull Model: In this model, Alertz accumulates all the notifications in memory instead of publishing it. These notifications are available for consumption from an http end point as a json.
Trends

In addition to notifications from nagios and mail, you can view graphs showcasing the health of a rule over a period of time.
